# Archived migrations (pre-baseline)

These 70 migrations were the project's original schema history. By the time the
live Supabase database was inspected, it had **drifted** away from them — the
deployed schema differs in ways that make these files unsafe to replay:

- `cargo_type_v2_enum` (here) is `cargo_type_enum` on the live DB
- `vessel_contacts` (here) → `vessel_contact_history` (live)
- `vessel_claims` (here) → `ownership_claims` (live)
- `review_queue` columns differ (`submitted_at`/`reviewed_at`/`admin_note` vs
  `created_at`/`updated_at`/`amendment_detail`)
- `users.role` is `text` (live) vs a `user_role` enum (here)
- live has extra tables not defined here: `matches`, `announcements`
- the canonical matching RPC referenced `va.ref`, a column the live
  `vessel_availability` does not have

They are kept **for reference/history only** and are intentionally outside the
active migrations sequence (the Supabase CLI ignores subdirectories).

## Source of truth

`supabase/migrations/20260616120000_remote_baseline.sql` is now the single
baseline, generated by introspecting the live database (enums, tables,
constraints, indexes, functions, views, triggers, RLS policies, grants).

Do **not** re-run these archived files against the live database.
